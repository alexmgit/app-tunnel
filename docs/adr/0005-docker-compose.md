# 0005 Docker Compose with Domain and SSL

Date: 2026-03-03

## Context

Нужно упростить продакшен запуск до одной команды и сразу закрыть типовой сценарий:
- публичный домен,
- wildcard поддомены для туннелей,
- автоматический TLS.

Отдельные `docker run` команды для сервера/прокси усложняют эксплуатацию и повышают риск рассинхрона конфигурации.

## Decision

- Добавить `docker-compose.yml` как основной способ деплоя.
- Использовать Caddy как внешний TLS-терминатор перед `app-tunnel-server`.
- Проксировать:
  - `control.<DOMAIN>` -> `app-tunnel-server:8080`,
  - `<subdomain>.<DOMAIN>` -> `app-tunnel-server:8082`.
- Хранить Caddy state в volume (`caddy_data`, `caddy_config`) для сохранения сертификатов.
- Оставить tunnel TCP порт (`8081`) опубликованным для клиентских соединений.

## Consequences

- Запуск и обновление стека выполняются одной командой `docker compose up -d --build`.
- TLS и выпуск сертификатов выносятся из приложения во внешний слой.
- Появляется явная зависимость от Caddy и DNS-настроек (`DOMAIN` и wildcard запись).
