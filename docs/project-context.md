# Project Context

Date: 2026-02-04

## Purpose

Минимальный ngrok-подобный сервис для HTTP/HTTPS туннелирования. Два бинарника: сервер и клиент. Фокус на простоте, надежности и использовании только Go stdlib.

## Core Capabilities

- Регистрация поддомена клиентом через контрольный API.
- Проксирование входящих HTTP/HTTPS запросов к локальному сервису клиента.
- Пул туннельных соединений для параллельной обработки.
- Переиспользование ранее выданных поддоменов.

## Non-Goals (v1)

- TCP/UDP raw туннелирование.
- Авторизация/мульти-tenant контроль доступа.
- Встроенный ACME/Let's Encrypt.
- HTTP/2 и WebSocket.

## Architecture Overview

- Server:
  - Control plane: `POST /register`.
  - Data plane: HTTP/HTTPS прокси по поддомену.
  - Tunnel listener: принимает клиентские соединения.
- Client:
  - Регистрирует поддомен.
  - Поднимает пул туннельных соединений к серверу.
  - Форвардит запросы к `LOCAL_FORWARD_ADDR`.

## Configuration

Только через переменные окружения. Все адреса и таймауты задаются в `.env.example`.

## Operational Notes

- TLS рекомендуется завершать внешним прокси (Caddy/Traefik/nginx+certbot).
- `SUBDOMAIN_STORE_PATH` хранит историю поддоменов для повторного использования.
- Логи в stdout/stderr; ротация на стороне окружения.
